# Examen Práctico - Backend 1
## Modelado inicial de un sistema para un local de comida rápida

## Propósito

Construir una solución inicial en **Java con Spring Boot** que permita representar, desde el backend, un problema de contexto real relacionado con la operación de un local de comida rápida.

---

## Contexto

Un emprendimiento local de comida rápida desea iniciar la digitalización de algunos procesos básicos de su negocio.  
Hasta ahora, gran parte de la información se maneja de forma informal, lo que dificulta el control de productos, pedidos, atención al cliente y registro de ventas.

La administración del local desea contar con una primera versión de un sistema que le permita organizar mejor la información de su operación diaria y sentar bases para futuras mejoras tecnológicas.

---

## Actividad

A partir del contexto descrito, usted debe analizar la situación y proponer una solución en **Spring Boot**.

Su trabajo consiste en identificar los elementos principales del dominio, convertirlos en clases del sistema y prepararlos para su almacenamiento en una base de datos **H2**.

---

## Condiciones del ejercicio

- Debe construir **mínimo 5 modelos o entidades**.
- Las clases deben surgir del análisis del problema.
- Cada clase debe incluir atributos definidos por usted de acuerdo con el contexto.
- En esta primera versión, las entidades **no deben estar relacionadas entre sí**.
- El proyecto debe ejecutarse correctamente.
- La persistencia debe realizarse sobre base de datos **H2**.

---

## Entregables esperados

Debe entregar un proyecto que incluya:

- Estructura base en Spring Boot.
- Entidades/modelos definidos a partir del problema.
- Persistencia en H2.

- Repositorio actualizado con su solución.
